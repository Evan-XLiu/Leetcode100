# 1. 两数之和 (Two Sum)

## 题目描述
给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出 **和为目标值** `target` 的那 **两个** 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。

你可以按任意顺序返回答案。

### 示例
* **示例 1：**
    * 输入：nums = [2,7,11,15], target = 9
    * 输出：[0,1]
    * 解释：因为 nums[0] + nums[1] == 9 ，返回 [0, 1] 。
* **示例 2：**
    * 输入：nums = [3,2,4], target = 6
    * 输出：[1,2]
* **示例 3：**
    * 输入：nums = [3,3], target = 6
    * 输出：[0,1]

### 限制
* $2 \le \text{nums.length} \le 10^4$
* $-10^9 \le \text{nums}[i] \le 10^9$
* $-10^9 \le \text{target} \le 10^9$
* **只会存在一个有效答案**

## 解法思路

这里提供两种常见的解法。在实际面试或开发中，我们通常推荐使用**哈希表法**，因为它利用空间换取时间，大幅提高了查找效率，体现了典型的“以空间换时间”的算法思想。

**解法一：暴力枚举（不推荐，时间复杂度较高）**
最容易想到的方法是采用双层循环，枚举数组中的每一个数 `x`，并在它后面的元素中寻找是否存在 `target - x`。虽然逻辑简单，但会产生大量的重复计算。

**解法二：哈希表（推荐，最优解）**
暴力法的时间复杂度较高的原因是寻找 `target - x` 的过程十分缓慢。我们可以使用哈希表（在 Python 中即为字典 `dict`）将已经遍历过的元素及其索引存储起来。这样，在后续遍历时，寻找 `target - x` 的时间复杂度就可以从 $O(N)$ 降低到 $O(1)$。

### 算法步骤
以下为**哈希表法**的具体执行步骤：
1.  创建一个空的字典 `num_map`，用于存储已经遍历过的数字及其对应的索引（键为数值，值为索引）。
2.  使用 `enumerate` 函数遍历数组 `nums`，每次循环获取当前元素的索引 `i` 和数值 `num`。
3.  计算当前数字所需的配对目标值 `complement = target - num`。
4.  检查 `complement` 是否已经作为键存在于 `num_map` 中：
    * 如果存在，说明我们找到了匹配的两个数。直接返回它们的索引：`[num_map[complement], i]`。
    * 如果不存在，将当前的数字 `num` 作为键，它的索引 `i` 作为值，存入 `num_map` 中，以便供后续的数字进行匹配。

### 复杂度分析
* **解法一（暴力枚举）：**
    * **时间复杂度:** O(N^2)，其中 N 是数组中的元素数量。最坏情况下，数组中任意两个数都要被尝试匹配一次。
    * **空间复杂度:** O(1)，只需要常数级别的额外空间。
* **解法二（哈希表）：**
    * **时间复杂度:** O(N)，其中 N 是数组中的元素数量。我们只遍历了包含有 N 个元素的数组一次。在字典中进行查找的时间复杂度为 O(1)。
    * **空间复杂度:** O(N)，其中 N 是数组中的元素数量。主要为字典需要分配的空间，最坏情况下（即直到数组最后一个元素才找到答案），字典中会存储 N - 1 个键值对。

## 代码实现

### python
```python
class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        """
        使用哈希表（字典）来降低时间复杂度，实现一次遍历即可找到答案。
        """
        # 初始化一个字典，用于存储数值及其对应的索引
        num_map = {}
        
        # 遍历列表，同时获取索引和数值
        for i, num in enumerate(nums):
            # 计算需要的另一个加数
            complement = target - num
            
            # 如果这个加数已经在字典中，说明找到了答案
            if complement in num_map:
                return [num_map[complement], i]
            
            # 如果不在字典中，将当前数值和索引存入字典
            num_map[num] = i
            
        # 根据题意，必定会有一个有效答案，正常情况下不会执行到这里
        return []
```

这道题中的key和value其实是相反的，把list中的value当作dict里面的key。因为dict在找key的时候是O(1)。

### 穷举
```python
class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        for i in range(len(nums)):
            for j in range(i+1, len(nums)):
                if nums[i] + nums[j] == target:
                    return [i, j]
```
